# RTT调试工具 - 使用说明

## 快速开始

### 1. 运行程序

双击 `dist/RTT-Tool.exe` 启动程序

### 2. 连接MCU

1. 确保JLink已连接到MCU
2. 确保MCU已移植RTT代码并初始化
3. 点击工具栏的"连接"按钮
4. 程序将自动连接并初始化RTT

### 3. 接收数据

- 接收到的数据会自动显示在接收区
- 勾选"时间戳"可显示接收时间
- 勾选"HEX显示"可查看HEX格式数据

### 4. 发送数据

1. 在发送区输入数据
2. 选择发送模式：
   - 字符串：发送文本数据
   - HEX：发送十六进制数据（如：01 02 03）
3. 勾选"加换行"可自动添加换行符
4. 点击"发送"按钮或按回车键

## 功能说明

### 工具栏功能

| 按钮 | 功能 |
|------|------|
| 连接 | 连接到MCU |
| 断开 | 断开连接 |
| 清空 | 清空接收区 |
| 时间戳 | 显示/隐藏时间戳 |
| HEX显示 | 切换HEX/字符串显示模式 |
| 字体 | 设置字体 |
| 置顶 | 窗口置顶 |

### 发送模式

- **字符串模式**：发送普通文本，支持UTF-8编码
- **HEX模式**：发送十六进制数据，支持空格分隔

## 注意事项

1. **JLink路径**：程序会自动查找JLink DLL，默认路径：
   - D:\Program Files\SEGGER\JLink_V938a\JLink_x64.dll (64位)
   - D:\Program Files\SEGGER\JLink_V930\JLinkARM.dll (32位)
   - 程序会根据Python位数自动选择正确的DLL

2. **JLink位数要求**：
   - **重要**: JLink DLL位数必须与Python位数匹配!
   - 如果使用64位Python,程序会自动查找JLink_x64.dll
   - 如果使用32位Python,程序会自动查找JLinkARM.dll
   - JLink V938a及以上版本同时包含32位和64位DLL
   - 程序会自动检查并提示位数不匹配问题

3. **MCU要求**：
   - MCU需要已移植SEGGER RTT代码
   - RTT需要已初始化
   - 建议RTT缓冲区大小 >= 1KB

4. **性能建议**：
   - 大量数据接收时建议使用HEX显示模式
   - 建议接口速度设置为4000kHz或更高
   - 建议使用SWD接口（速度更快）

## 配置文件

## 故障排除

### 问题1：无法找到JLinkARM.dll

**解决方案**：
1. 确认已安装JLink软件
2. 检查安装路径是否正确
3. 或修改代码中的JLink路径

### 问题2：JLink DLL位数不匹配

**错误提示**: "JLink DLL位数不匹配!"

**解决方案**：
1. 检查Python位数: `python -c "import struct; print(struct.calcsize('P') * 8)"`
2. 如果Python是64位,安装64位JLink软件
3. 如果Python是32位,安装32位JLink软件
4. JLink下载地址: https://www.segger.com/downloads/jlink/

### 问题3：连接失败

**解决方案**：
1. 检查JLink是否正确连接到MCU
2. 检查MCU是否已上电
3. 检查接口类型（SWD/JTAG）是否正确
4. 尝试降低接口速度

### 问题4：无数据显示

**解决方案**：
1. 确认MCU已初始化RTT
2. 确认MCU正在发送数据
3. 检查RTT缓冲区是否已满
4. 尝试重新连接
