## v1.2 更新内容

### 🎉 主要新增功能

1. **日志窗口**
   - 新增独立的日志窗口,显示连接、通讯和错误日志
   - 支持日志类型过滤(全部/INFO/WARNING/ERROR/SUCCESS)
   - 不同类型日志使用不同颜色显示
   - 支持清空日志功能
   - 工具栏新增"日志"按钮,可随时打开/关闭日志窗口

2. **瑞萨设备型号支持**
   - 设备型号列表新增: R9A07G084M04
   - 支持瑞萨(Renesas)系列MCU

3. **RTT控制块地址自动保存**
   - 手动输入RTT控制块地址后自动保存
   - 下次打开连接对话框时自动填充上次的地址
   - 配置文件新增rtt_address字段

### 📋 功能详情

| 功能 | 说明 |
|------|------|
| 日志窗口 | 显示连接、通讯、错误日志 |
| 日志类型 | INFO/WARNING/ERROR/SUCCESS |
| 日志颜色 | 绿色/黄色/红色/青色 |
| 日志过滤 | 支持按类型过滤 |
| 瑞萨设备 | R9A07G084M04 |
| RTT地址保存 | 自动保存和填充 |

### 🔧 技术改进

1. **新增LogService** (log_service.py)
   - 日志记录和管理服务
   - 支持多种日志类型
   - 最大保存1000条日志

2. **新增LogWindow** (log_window.py)
   - 独立的日志显示窗口
   - 支持日志过滤和清空
   - 使用不同颜色显示不同类型

3. **更新ConnectionService**
   - 集成日志服务
   - 连接过程记录详细日志
   - 错误信息记录到日志

4. **更新ConfigService**
   - 新增rtt_address配置项
   - 支持RTT地址持久化

5. **更新ConnectionDialog**
   - 支持RTT地址自动填充
   - 新增瑞萨设备型号

### 🎯 使用方法

1. **查看日志**
   - 点击工具栏"日志"按钮
   - 日志窗口显示所有操作日志
   - 可按类型过滤日志

2. **连接瑞萨设备**
   - 点击"连接"按钮
   - 在设备型号中选择或输入: R9A07G084M04
   - 配置其他参数后连接

3. **使用RTT地址保存**
   - 选择"地址"模式
   - 输入RTT控制块地址(如: 0x20000000)
   - 下次连接时自动填充该地址

## v1.1 更新内容

### 🔧 主要修复

1. **修复DLL加载问题**
   - 使用pylink-square库代替直接DLL调用
   - 自动查找JLinkARM.dll（支持多个路径）
   - 支持当前目录下的DLL

2. **新增连接配置对话框**
   - 完全模仿JLinkRTTViewer的连接界面
   - 支持USB/TCP/IP连接方式
   - 支持序列号/IP地址指定
   - 支持设备型号选择
   - 支持接口类型和速度设置
   - 支持RTT控制块地址配置（自动检测/指定地址/搜索范围）

3. **集成pylink库**
   - 使用pylink-square库（更稳定）
   - 支持所有JLink功能
   - 更好的错误处理

### 📋 连接对话框功能

| 功能 | 说明 |
|------|------|
| 连接方式 | USB / TCP/IP |
| 设备识别 | 序列号或昵称 |
| 目标设备 | 支持多种MCU型号 |
| 接口类型 | SWD / JTAG |
| 接口速度 | 1000-20000 kHz |
| RTT控制块 | 自动检测 / 指定地址 / 搜索范围 |

### 🎯 使用方法

1. **启动程序**
   ```bash
   python main.py
   ```
   或运行打包后的exe：
   ```bash
   dist/RTT-Tool.exe
   ```

2. **连接MCU**
   - 点击"连接"按钮
   - 在弹出的对话框中配置连接参数
   - 点击"确定"开始连接

3. **配置说明**
   - **连接方式**：选择USB或TCP/IP
   - **设备型号**：选择或输入MCU型号（如Cortex-M4）
   - **接口**：选择SWD或JTAG
   - **速度**：选择接口速度（建议4000 kHz）
   - **RTT控制块**：
     - 自动检测：让JLink自动查找RTT控制块
     - 地址：手动指定RTT控制块地址（如0x20000000）
     - 搜索范围：指定搜索范围
